# Strategy Narrative

Transform strategic content into narratives that move people to action.

## Core Principle

Strategy is getting people to do something. Without narrative, strategy is powerless—a diagram without words, a recommendation without momentum. The skill's job is to give strategy its persuasive shape.

## The Four-Layer Model

Every compelling strategy narrative operates on four layers simultaneously:

| Layer | Function | Test |
|-------|----------|------|
| **Spine** | Five-beat dramatic arc | Does each beat exist and connect seamlessly? |
| **Rhythm** | Contrast between "what is" and "what could be" | Does it oscillate to sustain tension? |
| **Edge** | Sharp, audience-specific arguments | Can someone repeat the core idea in one sentence? |
| **Craft** | Ruthless subtraction | Does every sentence create Movement or useful Colour? |

## Layer 1: Spine (The Five-Beat Arc)

Map every strategy narrative to this structure before writing anything:

```
REFLECTION → AMBITION → JEOPARDY → HOPE → SOLUTION
(Where we are)  (Where we could be)  (What blocks us)  (The crack of light)  (The idea that unlocks it)
```

**Reflection**: Ground the audience in shared reality. What's true today that everyone recognizes?

**Ambition**: Lift to possibility. What could tomorrow look like? Make it vivid and desirable.

**Jeopardy**: Plunge into stakes. What threatens us? What happens if we don't act? This is where tension lives—without it, solutions feel abstract.

**Hope**: Find the foothold. What insight, advantage, or truth gives us a way through? This is often where the strategic insight lives.

**Solution**: Land the idea. What specifically do we do? This earns its power from everything before it.

**The Nelson Test**: Can you reduce the entire narrative to five sentences, one per beat? If not, the spine isn't clear.

Example (Battle of Trafalgar):
1. We're facing the combined Franco-Spanish fleet. (Reflection)
2. We must destroy the enemy fleet. (Ambition)
3. We're massively outnumbered—if we lose, French rule within weeks. (Jeopardy)
4. Their gunners are less trained than ours. (Hope)
5. We'll sail directly at them to divide their line and force chaos. (Solution)

## Layer 2: Rhythm (The Sparkline)

Within each beat, oscillate between "what is" (current reality) and "what could be" (future possibility).

**Why this matters**: A flat "Report" shape (long on reality, quick recommendation at end) is boring. A flat "Pitch" shape (quick setup, long on benefits) feels like a sales trap. The Sparkline earns its ending by making the audience feel the gap repeatedly.

**Pattern**:
```
What is → What could be → What is → What could be → ... → New Bliss
```

**Practical application**: After stating a problem, immediately show what solving it would feel like. After painting possibility, drop back to the obstacle. This contrast sustains attention.

## Layer 3: Edge (Sharp Arguments)

### The One-Sentence Test
Before writing, answer: "What is the one sentence you want people repeating to others after this?"

If you can't answer clearly, stop. The narrative isn't ready.

**Topic vs. Idea**:
- "Digital transformation" = topic (category, no stakes)
- "Most digital transformations fail because they start with technology, not behavior" = idea (claim with stakes)

Force the input toward an idea, not a topic.

### Audience as Hero
Position the audience as the hero facing a challenge. The strategist is the mentor providing tools. Never make the strategy about how clever the presenter is.

### Evidence That Lands
- Few sharp data points beat floods of information
- Specificity over volume—what motivates *this* audience?
- Stories carry messages further than facts (but stories must reinforce the central idea or get cut)

## Layer 4: Craft (Ruthless Subtraction)

### Movement vs. Colour
Every sentence must do one of two things:

**Movement**: Advances the narrative to the next beat.
> "The house is on fire!" → "I'm calling the fire brigade!"

**Colour**: Adds useful context without advancing.
> "The house is on fire!" → "I'm scared!"

Movement without Colour = unexplained assertions.
Colour without Movement = useless tone poems.

Interrogate every sentence: "Is this Movement or Colour? If neither, delete."

### The Direction Test
Stories travel forward only. Read the narrative aloud. Any point that moves backward or sideways? Eliminate it.

### Small Words for Big Ideas
Jargon signals membership but obscures truth. Use words you'd use in conversation. If a sentence needs rereading, rewrite it.

Test: Could someone explain your idea to a friend in normal language after hearing it once?

## Workflow

### When Transforming Raw Content:

1. **Extract the core idea** (one sentence the audience should repeat)
2. **Map to the five-beat spine** (find or force each beat)
3. **Identify what's missing** (usually Jeopardy is underdeveloped—tension creates permission for solutions)
4. **Rewrite each beat with contrast** (oscillate between what is / what could be)
5. **Cut ruthlessly** (Movement or Colour only; forward direction only)
6. **Test aloud** (does it flow? does each point lead to the next?)

### When Assessing Existing Strategy:

1. Can you extract a clear one-sentence idea?
2. Do all five beats exist? Which are weak?
3. Is there sufficient Jeopardy to earn the Solution?
4. Does it oscillate or run flat?
5. Does every sentence pull its weight?

## Anti-Patterns to Fix

| Problem | Symptom | Fix |
|---------|---------|-----|
| No spine | Feels like a list of points | Map to five beats; find the missing ones |
| Jeopardy deficit | Solutions feel abstract, forgettable | Spend more time making the audience feel the problem |
| Flat rhythm | Monotonous, audience drifts | Add contrast—after every "what is," show "what could be" |
| Topic not idea | No stakes, nothing to disagree with | Sharpen to a claim someone could argue against |
| Movement starvation | Lots of Colour, no forward pull | Ask "what happens next?" after each sentence |
| Conclusion-leaping | Audience doesn't follow the logic | Show how each link connects; hold their hand |

## Output Guidance

When producing the transformed narrative:

- Write in prose, not bullets (unless explicitly requested)
- Keep language conversational—small words, clear sentences
- Make the spine visible but not mechanical
- Front-load the stakes; earn the solution
- End with "new bliss"—a vivid, attainable future state

The goal is a narrative someone could deliver from memory, that moves through the room like electricity, and that people repeat to others afterward.

## Quality Gate (MANDATORY before presenting)

Run two checks:

**Narrative structure check:**
| Check | Pass/Fail |
|-------|-----------|
| All five beats exist (Reflection, Ambition, Jeopardy, Hope, Solution) | |
| Jeopardy is developed enough that the Solution feels earned | |
| One-sentence test passes — the core idea can be repeated in one sentence | |
| Every sentence is Movement or Colour — nothing is neither | |
| Narrative reads forward only — no backward or sideways moves | |

**Voice check:**
Run Voice DNA + Stop-Slop from `rules/voice.md` line by line on the output. Small words for big ideas. No jargon. No passive constructions. Every sentence earns the next.

If Jeopardy is underdeveloped or the one-sentence test fails, revise before presenting.
