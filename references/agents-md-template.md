# AGENTS.md Template

Generated by `lead pipeline` when initializing a new project.
Follows the AGENTS.md Linux Foundation standard (60K+ projects, Dec 2025).

## Template

```markdown
# AGENTS.md

## Project
[Project name]: [One sentence description from the brief's rallying cry + proposition]

## Rallying Cry
[2-5 word core statement from the brief]

## Architecture
[Tech stack, framework, key dependencies. Populated from intake constraints.]

## Conventions
- [Code style: e.g., "TypeScript strict mode, Biome for formatting"]
- [Component patterns: e.g., "React Server Components by default, client only when needed"]
- [File organization: e.g., "Feature-based folders under src/features/"]

## Commands
```bash
# Development
[dev command]

# Build
[build command]

# Test
[test command]

# Lint
[lint command]
```

## Strategy Context
- Brief: `docs/strategy/04-brief.md`
- Spec: `docs/strategy/05-spec.md`
- Component map: `docs/map/components.md`
- Vision (agent north star): `docs/vision.md`

## Non-Goals (DO NOT BUILD)
[Carried from the brief's "What We're NOT Doing" section.
This is the most important section for agents. It prevents scope creep
across every tool that reads this file.]
```

## Generation Rules

1. The template is populated from existing project artifacts:
   - Rallying cry and non-goals from `docs/strategy/04-brief.md`
   - Tech stack from `docs/strategy/01-intake.md` constraints
   - Commands detected from `package.json`, `Makefile`, `pyproject.toml`, etc.

2. If no brief exists yet (pipeline just started), generate a minimal AGENTS.md
   with just the project name and tech stack. Update it after the brief is locked.

3. The Non-Goals section must always be present. If the brief doesn't exist,
   leave it as a placeholder: `[To be populated from the strategic brief]`.

4. AGENTS.md lives at project root. It is tool-agnostic (read by Claude, Cursor,
   Copilot, Codex, and any future agent tool).
